# 🗄️ MongoDB Production Setup Guide

Complete guide to set up MongoDB for production deployment.

---

## ✅ Option 1: MongoDB Atlas (Recommended - Free Tier Available)

MongoDB Atlas is the easiest and most popular option. You're already using it!

### Step 1: Create/Login to MongoDB Atlas Account

1. Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)
2. Sign up (free) or log in
3. Create a new organization (if first time)

### Step 2: Create a New Cluster (or Use Existing)

#### For Production:
1. Click **"Create"** or **"Build a Database"**
2. Choose deployment option:
   - **M0 (Free Tier)** - Good for testing/small apps
   - **M10+ (Paid)** - Better for production (recommended)
3. Select **Cloud Provider**: AWS, Google Cloud, or Azure
4. Select **Region**: Choose closest to your Render region
5. Click **"Create Cluster"**
6. Wait 3-5 minutes for cluster to be created

### Step 3: Create Database User

1. Go to **"Database Access"** (left sidebar)
2. Click **"Add New Database User"**
3. Choose authentication method:
   - **Password** (recommended)
4. Enter:
   - **Username**: `bottleji-prod` (or your choice)
   - **Password**: Generate a strong password (click "Autogenerate Secure Password" or create your own)
   - **Save the password!** You'll need it for connection string
5. Set user privileges:
   - **Built-in Role**: `Atlas admin` (for full access)
   - Or `Read and write to any database` (more secure)
6. Click **"Add User"**

### Step 4: Whitelist IP Addresses

1. Go to **"Network Access"** (left sidebar)
2. Click **"Add IP Address"**
3. For production, you have two options:

   **Option A: Allow All (Easier - for now)**
   - Click **"Allow Access from Anywhere"**
   - IP Address: `0.0.0.0/0`
   - Click **"Confirm"**
   - ⚠️ Less secure, but easier for deployment

   **Option B: Render IP Ranges (More Secure)**
   - Render's IP ranges change, so Option A is easier
   - Or add specific IPs if you know them

### Step 5: Get Connection String

1. Go to **"Database"** (left sidebar)
2. Click **"Connect"** on your cluster
3. Choose **"Connect your application"**
4. Select:
   - **Driver**: Node.js
   - **Version**: 5.5 or later
5. Copy the connection string

**Format:**
```
mongodb+srv://<username>:<password>@cluster0.xxxxx.mongodb.net/<database-name>?retryWrites=true&w=majority
```

### Step 6: Customize Connection String

Replace these parts:
- `<username>` → Your database username (from Step 3)
- `<password>` → Your database password (from Step 3)
- `<database-name>` → `bottleji` or `eco_collect` (your choice)

**Example:**
```
mongodb+srv://bottleji-prod:MySecurePassword123@cluster0.xxxxx.mongodb.net/bottleji?retryWrites=true&w=majority
```

### Step 7: Test Connection

Test locally first:
```bash
# In your backend directory
export MONGODB_URI="your-connection-string-here"
npm run start:dev
# Check logs - should connect successfully
```

## 🔒 Production Best Practices

### 1. Separate Production Database
- ✅ Create separate cluster for production
- ✅ Or at least separate database
- ❌ Don't use same database as development

### 2. Strong Passwords
- ✅ Use autogenerated secure passwords
- ✅ Store passwords securely (in Render env vars)
- ❌ Don't use simple passwords

### 3. IP Whitelisting
- ✅ For production: Allow `0.0.0.0/0` (all IPs) initially
- ✅ Later: Restrict to Render's IP ranges if possible
- ⚠️ Note: Render IPs can change

### 4. Database Backups
- ✅ Enable automated backups in MongoDB Atlas
- ✅ Set up backup schedule
- ✅ Test restore process

### 5. Monitoring
- ✅ Enable MongoDB Atlas monitoring
- ✅ Set up alerts for:
  - High connection count
  - Slow queries
  - Storage usage

## 🚀 Add to Render

Once you have your production MongoDB connection string:

1. Go to Render → Your Service → Environment
2. Add environment variable:
   - **Key**: `MONGODB_URI`
   - **Value**: `mongodb+srv://username:password@cluster.mongodb.net/database?retryWrites=true&w=majority`
3. Save and redeploy

## 🔍 Troubleshooting

### "Authentication failed"
- Check username and password are correct
- Verify password doesn't have special characters that need encoding
- Test connection string locally first

### "IP not whitelisted"
- Go to Network Access in MongoDB Atlas
- Add `0.0.0.0/0` to allow all IPs
- Wait 1-2 minutes for changes to propagate

### "Connection timeout"
- Check cluster is running (not paused)
- Verify connection string is correct
- Check network access settings

### "Database not found"
- Database is created automatically on first connection
- Or create it manually in MongoDB Atlas
- Verify database name in connection string
